Technical Blog - Classes vs ID's

Introduction to HTML and CSS

When working with HTML / CSS (two out of the three main building blocks for websites) we want to be able to style our raw tags inside HTML using three different methods. The first method is to use the 'style' tag inside the HTML directly. This works the same as linking our HTML to a CSS file except if working with a large piece of code it can get messy and we can easily miss things when trying to get our HTML code to look a specific way. The easier method is to link our HTML and CSS together using a 'link' tag with the appropriate "href" which links to our CSS file that we created earlier. Once we have linked the HTML and CSS together, anything we do on the CSS file will apply to our HTML.


But how does the HTML know what CSS rules to apply? We can figure that out using the other two methods I mentioned before, classes and ID's.

We can create classes and ID's in CSS using a specific syntax for each and then writing rules for those classes/ID's in the code block for that specific class or ID.

Lets discuss ID's now.

When to use classes vs ID's?

In the browser, classes have no special function. Their main purpose is to allow you to apply styles to a range of elements on a web page. IDs, on the other hand, can be used by the browser to navigate to a certain part of the web page. If you assign an element an ID, you can use a special URL to link directly to that element. This behavior works because IDs are unique on a web page. Source "https://careerkarma.com/blog/css-class-vs-id-2/#:~:text=an%20ID%20selector.-,Conclusion,to%20apply%20to%20multiple%20elements."

Conclusion

So in closing, classes and ID's can both be used to apply styling from a CSS file to an HTML file. Classes are non-unique meaning they can be used alot over different parts of your HTML as much as you like. ID's are unique meaning they can only be used once per webpage but they also have special functionality allowing you to assign special URL's to them which link them to other elements on the page meaning if you click on a button that says "Contact Information" it will pull you to the contact information section on that webpage.
